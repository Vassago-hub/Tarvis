# M8 决策清单与拍板项

## 1. 代码落点

推荐选择：先放 `examples/tcl-official-site`。

| 方案 | 优点 | 缺点 | 建议 |
| --- | --- | --- | --- |
| `examples/tcl-official-site` | 轻量、低侵入、适合验证 | 暂时不像正式包 | MVP 推荐 |
| `packages/tcl-agent` | 更正式、可发布、结构清晰 | 前期维护成本高 | 验证稳定后再迁移 |
| 直接改 `page-agent` | 快 | 污染通用框架 | 不推荐 |

拍板建议：先 example，灰度稳定后升 package。

## 2. UI 形态

推荐选择：客服式浮窗 + 保留开发者面板。

| 方案 | 建议 |
| --- | --- |
| 当前 Panel 直接改 | 不推荐 |
| 新增 CustomerServicePanel | 推荐 |
| 接 TCL 现有客服 UI | 如果公司已有统一客服组件，可后续对接 |

拍板建议：先新增或演化通用客服 UI，不破坏当前调试能力。

## 3. Agent 安全机制

推荐选择：代码强制拦截 + prompt 约束。

不能只靠 prompt。

必须拍板：

- 外链前确认。
- 提交前确认。
- 隐私字段不静默填写。
- 登录、验证码、支付停止。
- 生产禁用 `execute_javascript`。

拍板建议：全部作为 P0。

## 4. 知识库方式

推荐选择：先静态 FAQ + 官网路径规则，后接 RAG。

| 方案 | 优点 | 缺点 |
| --- | --- | --- |
| 静态 FAQ | 稳、快、可审核 | 覆盖有限 |
| 官网 RAG | 覆盖广 | 需要抓取、清洗、召回评估 |
| 内部客服知识库 | 最有价值 | 需要权限和合规 |

拍板建议：MVP 用静态 FAQ，30 个高频问题；RAG 放第二阶段。

## 5. 客服系统对接

推荐选择：第一阶段不深度对接，只打开官方入口。

| 方案 | 建议 |
| --- | --- |
| 打开在线客服入口 | MVP 推荐 |
| 传递上下文给客服系统 | 第二阶段 |
| 自动建工单 | 暂不做 |
| 自动提交投诉 | 不做 |

拍板建议：AI 可整理问题摘要，用户确认后自行提交或复制给人工。

## 6. 意图识别方式

推荐选择：规则优先 + LLM 兜底。

第一阶段意图少，不必上复杂分类模型。

优先级：

```text
complaint > human_support > repair_service > troubleshooting > business_service > site_navigation > unknown
```

拍板建议：先规则命中关键词，再让 LLM 处理模糊输入。

## 7. 官网操作范围

推荐选择：只操作 TCL 官网当前域名页面。

允许：

- `https://www.tcl.com/cn/zh`
- 官网服务页、产品页、文章页。

确认后允许：

- 官方在线客服外链。
- 官方商城、京东等购买外链。

禁止：

- 支付页。
- 登录页。
- 第三方未知页面。
- 自动跨站连续操作。

## 8. 监控与日志

推荐选择：上线前必须有最小 telemetry。

必须记录：

- 意图。
- 风险等级。
- 是否确认。
- 成功或失败。
- 转人工原因。
- 页面 URL。
- 延迟。
- 用户反馈。

必须脱敏：

- 手机号。
- 地址。
- 订单号。
- 身份证。
- 支付信息。

## 9. 成功指标

建议拍板为：

| 指标 | MVP 目标 |
| --- | --- |
| 找入口成功率 | 不低于 80% |
| 自助解决率 | 不低于 30% |
| 高风险确认率 | 100% |
| P0 误操作 | 0 |
| 负反馈率 | 不高于 5% |
| 首响 | 不高于 3 秒 |

## 10. 第一阶段不做事项

建议明确写入范围外：

- 不自动提交维修、投诉、预约。
- 不自动登录。
- 不处理支付。
- 不承诺价格、库存、维修时效。
- 不承诺赔偿结果。
- 不做全量商品导购。
- 不接复杂内部系统。
- 不替代人工客服所有职责。

## 最终推荐拍板版本

```text
项目名称：TCL 智能服务助手

MVP 范围：
- 售后降本
- 全站导航
- 人工客服分流
- 高频故障排查

代码落点：
- examples/tcl-official-site

UI：
- 新增或演化客服式浮窗
- 保留开发者 Panel 能力

知识：
- 静态 FAQ + 官网路径规则

安全：
- 高风险动作代码拦截
- 生产禁用 JS 执行
- 隐私字段用户手动

上线：
- 内部试用 -> 服务页灰度 -> 首页低比例灰度
```
