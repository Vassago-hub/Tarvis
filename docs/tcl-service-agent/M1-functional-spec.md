# M1 功能规格

## 目标

M1 定义第一阶段 MVP 的功能边界和用户流程。核心目标是用 AI 服务助手承接官网上的高频、低风险、重复咨询，并把复杂问题更高质量地转给人工客服。

## 1. 转人工客服

### 用户说法

- 我要人工。
- 联系客服。
- 在线客服在哪。
- 打客服电话。

### 流程

1. 识别为 `human_support`。
2. 回复用户可以打开 TCL 官方在线客服入口，也可以提供客服电话。
3. 优先打开服务支持页。
4. 寻找并点击 `24小时在线客服`。
5. 如果在线客服打不开、跨域、弹窗失败，提供客服电话 `4008-123456`。
6. 如果用户表达强烈不满或投诉，直接进入人工优先模式。

### 页面动作

- 可自动点击服务支持。
- 可自动点击在线客服入口。
- 如果将跳转外部客服系统，点击前提示用户确认。

### 风险等级

中。

### 禁止事项

- 不冒充人工客服。
- 不承诺排队时间。
- 不替用户编造投诉内容。

## 2. 报修与售后服务

### 用户说法

- 我要报修电视。
- 空调坏了。
- 冰箱维修。
- 洗衣机售后。
- 怎么预约维修。

### 流程

1. 识别为 `repair_service`。
2. 询问产品类型：电视、空调、冰箱、洗衣机、门锁、其他。
3. 如果用户已经说明产品，跳过询问。
4. 打开服务页。
5. 引导到家电服务或个人产品服务入口。
6. 如果页面要求手机号、地址、订单号，由用户自己输入或确认。
7. 提交前必须确认。

### 页面动作

- 可打开服务支持。
- 可点击家电服务、个人产品服务。
- 可帮助定位表单字段。
- 可代填非敏感文本草稿，例如产品型号，但提交前确认。

### 风险等级

中到高。

### 必须确认

- 输入手机号。
- 输入地址。
- 输入订单号。
- 提交预约或报修单。

### 兜底话术

```text
我已经帮你找到 TCL 官方售后入口。后续需要填写手机号、地址或订单号，为保护隐私，请你手动确认后再提交。
```

## 3. 常见故障排查

### 用户说法

- 空调 E1 怎么办。
- 电视黑屏。
- 冰箱声音大。
- 洗衣机不脱水。
- 遥控器没反应。

### 流程

1. 识别为 `troubleshooting`。
2. 判断产品类型和故障类型。
3. 给出安全优先的 3 到 5 步排查。
4. 推荐对应官网教程或服务入口。
5. 如果涉及安全风险，直接建议断电并联系售后。
6. 用户要求继续处理时，再打开服务支持入口。

### 示例话术

```text
可以先做几步安全排查：确认电源、遥控器/插座、滤网或连接线状态。如果出现异味、漏电、冒烟、进水，请立即断电并联系 TCL 官方售后。
```

### 风险等级

低到中。

### 高风险关键词

- 漏电。
- 烧焦味。
- 冒烟。
- 进水。
- 自行拆机。
- 压缩机异常。
- 电路板。
- 燃气。
- 高温。

### 高风险升级话术

```text
这个情况可能涉及安全风险，我不建议继续自行操作。请先断电，并联系 TCL 官方售后或人工客服。
```

## 4. 全站导航

### 用户说法

- 帮我找电视。
- 空调页面在哪。
- 智能门锁有哪些。
- 我要看会员俱乐部。
- 找商务合作。
- TCL+ APP 在哪。

### 流程

1. 识别为 `site_navigation`。
2. 判断目标栏目。
3. 如果目标明确，直接导航。
4. 如果目标模糊，问一个澄清问题。
5. 到达目标页后，告诉用户已打开。

### 页面动作

- 可点击首页导航。
- 可滚动查找页面区域。
- 可打开产品分类页。
- 可打开服务支持、会员俱乐部、商务合作等栏目。

### 风险等级

低。

### 兜底话术

```text
我没有在当前页面找到明确入口。我可以改为打开 TCL 服务支持页，或给你相关栏目名称让你手动选择。
```

## 5. 商用、光伏、合作分流

### 用户说法

- 光伏售后。
- 商用显示维修。
- 中央空调服务。
- 工程产品服务。
- 我要加盟。
- 商务合作。

### 流程

1. 识别为 `business_or_special_service`。
2. 区分服务类型：光伏、商用显示、中央空调、工程产品、合作加盟。
3. 打开服务页或对应导航。
4. 如果需要提交项目信息、联系方式、公司信息，必须确认。
5. 复杂问题建议转人工或商务团队。

### 风险等级

中到高。

### 禁止事项

- 不承诺合作条件。
- 不承诺报价。
- 不代替官方确认资质。
- 不在确认前提交公司或联系人信息。

## 需要新增的能力

| 模块 | 作用 |
| --- | --- |
| `IntentRouter` | 将用户输入分到客服、报修、故障、导航、商用合作等意图 |
| `TclInstructions` | TCL 专属系统规则和页面规则 |
| `SafetyPolicy` | 判断是否涉及隐私、提交、支付、投诉、安全风险 |
| `ConfirmBeforeAction` | 高风险动作前强制询问用户 |
| `SupportHandoff` | 失败或高风险时转人工 |
| `Telemetry` | 记录意图、路径、成功率、失败原因 |
| `KnowledgeAdapter` | 接入官网文章、服务页、产品页内容 |

## 优先开发顺序

1. TCL 专属 prompt 和规则。
2. 官网路径规则。
3. 高风险确认机制。
4. 客服式 UI。
5. 意图路由。
6. 埋点。
7. 知识库检索。
