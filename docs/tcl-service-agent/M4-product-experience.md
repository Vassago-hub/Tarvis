# M4 产品体验稿

## 产品名称

建议暂定为：TCL 智能服务助手。

不要对外叫 Page Agent 或 AI Agent。官网用户不关心技术名，应使用服务型、可靠型表达。

## 入口形态

右下角悬浮入口：

```text
[耳机图标] 智能客服
```

打开后：

```text
TCL 智能服务助手

你好，我可以帮你查找服务入口、排查常见故障、联系人工客服，也可以帮你在官网找到产品和服务页面。

[我要报修] [联系人工] [故障排查] [找产品]
```

第一屏不要解释太多技术能力。不要说“我可以帮你点击网页”。用户提出需求后再展示操作能力。

## 消息类型

客服 UI 建议只有 6 类消息：

| 类型 | 用途 |
| --- | --- |
| 用户消息 | 用户输入 |
| 助手回复 | 普通回答 |
| 操作状态 | 正在帮你打开服务支持 |
| 确认卡片 | 外链、提交、敏感信息前确认 |
| 转人工卡片 | 电话、在线客服、服务页入口 |
| 失败兜底 | 页面打不开、找不到、验证码、登录 |

不展示模型思考、工具名、元素 index、raw history。

现阶段如果保留开发者式操作展示，应默认收纳在右下角展开区，不占据页面中心。

## 状态文案

| 当前动作 | 用户看到 |
| --- | --- |
| 思考 | 正在理解你的问题... |
| 查页面 | 正在查看当前页面... |
| 点击导航 | 正在帮你打开服务支持... |
| 滚动查找 | 正在查找相关入口... |
| 等待加载 | 页面加载中，请稍等... |
| 完成 | 已为你打开相关页面。 |
| 失败 | 我没有在当前页面找到明确入口。 |

## 确认卡片

### 外链确认

```text
即将打开 TCL 官方在线客服页面

这可能会跳转到新的客服页面。
[继续打开] [取消]
```

### 提交确认

```text
即将提交服务申请

请确认手机号、地址、产品信息无误。提交后信息将发送给 TCL 官方服务系统。
[确认提交] [先不提交]
```

### 敏感输入确认

```text
这里需要填写手机号/地址等个人信息

为保护隐私，建议你手动填写。我可以停在这个表单位置。
[我来填写] [继续说明]
```

## 转人工卡片

用于人工客服、投诉、复杂维修、高风险故障。

```text
建议联系 TCL 官方客服

客服电话：4008-123456
服务时间：24小时在线客服入口以官网页面为准

[打开在线客服] [打开服务支持页]
```

投诉场景：

```text
这个问题更适合由 TCL 官方人工客服处理。我可以帮你打开官方客服入口，并把你刚才描述的问题整理成一段文字，方便你提交。
[整理问题] [打开人工客服]
```

## 典型流程

### 联系人工

```text
用户：我要人工客服

助手：我可以帮你打开 TCL 官方在线客服入口，也可以提供客服电话。
操作状态：正在打开服务支持页...

确认卡：即将打开 TCL 官方在线客服页面
[继续打开] [取消]

助手：已为你打开在线客服入口。如页面未正常打开，也可以拨打 4008-123456。
```

### 空调故障

```text
用户：空调显示 E1 怎么办？

助手：E1 通常需要结合具体机型判断。你可以先做安全排查：
1. 关闭空调电源，等待几分钟后重启。
2. 检查过滤网和进出风口是否堵塞。
3. 确认室内外机是否有异常声音或异味。
4. 如果反复出现 E1，建议联系 TCL 官方售后。

是否需要我帮你打开报修入口？
[打开报修入口] [联系人工] [不用]
```

### 报修

```text
用户：我要报修洗衣机

助手：可以。我先帮你打开 TCL 官方家电服务入口。
操作状态：正在查找家电服务入口...

助手：已找到相关服务入口。后续可能需要填写手机号、地址、产品型号等信息，为保护隐私，请你确认后再提交。
```

### 全站导航

```text
用户：帮我找智能门锁

助手：正在为你查找 TCL 智能门锁相关页面...
操作状态：正在打开个人及家庭产品...

助手：已为你打开智能门锁相关入口。你可以在这里查看产品系列和详情。
```

### 投诉

```text
用户：我要投诉，维修一直没人联系

助手：抱歉给你带来不好的体验。这个问题建议直接联系 TCL 官方人工客服处理。我可以帮你整理一段投诉说明，并打开官方客服入口。

投诉说明草稿：
“我已提交维修需求，但一直没有工作人员联系，希望尽快核实并反馈处理进度。”

[复制说明] [打开人工客服] [继续补充]
```

## 视觉风格建议

TCL 官网应走可信、清爽、服务感。

- 主色使用 TCL 品牌红作强调色。
- 大面积背景用白色或浅灰。
- 卡片圆角控制在 8px 内。
- 动作按钮清晰区分主次。
- 不使用炫光、渐变球、复杂动效。
- 移动端使用底部抽屉或右下角紧凑面板，桌面右下浮窗。
- TCL Ray 可作为助手形象展示，但不能遮挡输入框或关键页面内容。

## 客服助手人格

语气像官方服务人员：

- 简洁。
- 有边界。
- 不夸张。
- 不承诺无法确认的信息。
- 遇到安全问题优先提醒。
- 失败时给下一步。

推荐口吻：

```text
我可以帮你查找官方服务入口。
这个情况可能涉及安全风险，建议先断电并联系官方售后。
我没有在当前页面找到明确入口，可以为你打开服务支持页。
提交前请确认信息无误。
```

避免口吻：

```text
放心，肯定能修好。
这个价格一定最低。
我已经帮你提交了。
不用担心，我可以处理所有问题。
```
